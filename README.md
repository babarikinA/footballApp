# Footbool Landing (React + Vite)

Минималистичный лендинг футбольной платформы с маршрутизацией и «page object» структурой (страницы вынесены в отдельные компоненты).

## Стек
- React 19 + Vite 8
- React Router DOM (клиентские маршруты `/`, `/about`, `/contacts`)
- Чистый CSS (без Tailwind)
- Локальное хранилище (localStorage) как простая БД для примеров регистрации (имя, фамилия, пароль)

## Скрипты
- `npm run dev` — локальная разработка (Vite dev server)
- `npm run build` — продакшн сборка
- `npm run preview` — предпросмотр собранной версии
- `npm run lint` — линт проекта

## Структура
```
src/
  App.jsx               # точка входа маршрутов
  components/
    Header.jsx          # общий хедер + навигация
    HeroVisual.jsx      # визуальный мок экрана/поля
  data/
    content.js          # данные (фичи, шаги, статистика)
  pages/
    Home.jsx            # главная
    About.jsx           # о платформе
    Contacts.jsx        # контакты
    Register.jsx        # простая регистрация (имя/фамилия)
    Admin.jsx           # личный кабинет (пользовательский) и админ-панель
  App.css               # основная стилизация
  index.css             # базовый reset/behaviors
  data/
    content.js          # данные (фичи, шаги, статистика)
    db.js               # абстракция над localStorage (листинг/добавление/удаление)
```

## Page Object подход
- Каждая страница вынесена в отдельный компонент в `src/pages`.
- Общие элементы (хедер, визуальный блок) живут в `src/components`.
- Контентные данные хранятся в `src/data/content.js`, что упрощает наполнение без правок верстки.

## Быстрый старт
1) `npm install`
2) `npm run dev`
3) Открыть адрес, который покажет Vite (обычно `http://localhost:5173`).

## Куда править текст/контент
- Фичи, шаги и статистика: `src/data/content.js`
- Тексты блоков на страницах: `src/pages/*.jsx`
- Стили/отступы/цвета: `src/App.css`

## Регистрация и кабинеты (демо-БД)
- Регистрация: `/register` — имя/фамилия/пароль сохраняются в `localStorage` (пароль хэшируется base64 для демо).
- Кабинет игрока: `/admin` — вход по имени+паролю; профиль, друзья, команда, предстоящие матчи, статистика, загрузка фото. Получает уведомления о голосованиях от админа и может принять/отклонить.
- Админ: вход `admin` / `babasha`; вкладки «Настройки профиля», «Пользователи», «Голосования». Можно создавать голосования (дата, время, цена, игроки, приз, газон, освещение, раздевалка) и видеть таблицу ответов «за/против».
- Хранение: `src/data/db.js` — localStorage для пользователей, фото, голосований и ответов (без серверной БД).
